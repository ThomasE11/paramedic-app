# Site Visit Quick Reference Card

## 🚀 Quick Start (3 Steps)

1. **Timetables** → **Clinical Placements** → Find your site → Click **"Site Visit"**
2. **Fill out form** → General notes + Individual student feedback
3. **Save** → Notes automatically appear in each student's profile!

---

## 📝 Site Visit Checklist

### Before the Visit
- [ ] Check placement calendar for students at site
- [ ] Review previous visit notes
- [ ] Prepare discussion topics

### During the Visit
- [ ] Talk to clinical mentor
- [ ] Observe each student
- [ ] Document mentor feedback
- [ ] Note any concerns

### After the Visit (In the App)
- [ ] Record visit date/time
- [ ] Add general observations
- [ ] Complete feedback for each student
- [ ] Set follow-up if needed
- [ ] SAVE!

---

## 👤 Per-Student Feedback Fields

### Required
✅ **Performance** - Select rating
✅ **Attendance** - Present/Absent/Late/Left Early

### Highly Recommended
⭐ **Strengths** - What they did well
📈 **Areas for Development** - What to improve
💬 **Mentor Comments** - Feedback from site
✏️ **Skills Assessed** - What skills they practiced

### Optional
- Student Reflection
- Grade
- Hours Completed
- Specific Concerns
- Action Items

---

## 💾 What Gets Saved

When you click **"Save Site Visit"**:

1. ✅ Site visit record created
2. ✅ Individual notes for each student
3. ✅ **AUTO: Notes added to student profiles**
4. ✅ Categorized as "site_visit"
5. ✅ Timestamped and searchable

---

## 🔍 Finding Site Visit Notes

### In Student Profiles:
1. Go to **Students** section
2. Click on student name
3. Scroll to **Notes** section
4. Look for "Site Visit - [Site Name]"

### Filter Notes:
- By category: "site_visit"
- By date range
- By site/location

---

## ⌨️ Keyboard Shortcuts

- **Enter** after typing skill → Adds skill to list
- **Tab** → Move to next field
- **Ctrl/Cmd + S** → Save (in most browsers)

---

## 🎯 Tips for Efficient Site Visits

1. **Use Templates** - Keep common feedback phrases in notes app
2. **Skills List** - Have pre-written skills ready to copy/paste
3. **Batch Similar Feedback** - If multiple students need same development area
4. **Take Notes First** - Use paper/phone, transfer to app after
5. **Save Frequently** - Don't lose your work!

---

## 📊 Common Performance Ratings

- **Excellent** - Exceeds expectations, independent practice
- **Good** - Meets expectations, minor guidance needed
- **Satisfactory** - Acceptable performance, moderate supervision
- **Needs Improvement** - Below expectations, requires intervention
- **Unsatisfactory** - Significant concerns, immediate action needed

---

## ⚠️ When to Flag Concerns

Use the **Concerns** field for:
- Safety issues
- Professionalism problems
- Consistent skill deficits
- Mentor concerns
- Student wellbeing issues

Use **Action Items** for:
- Follow-up meetings
- Additional training
- Remediation plans
- Next visit focus areas

---

## 🔧 Troubleshooting

| Issue | Solution |
|-------|----------|
| Student not in list | Check they're assigned to this site |
| Can't find site | Use "Upload Schedule" or add manually |
| Notes didn't save | Check internet connection, try again |
| Wrong student selected | Remove and add correct one |

---

## 📱 Mobile Use

While we recommend desktop for full visits, you can:
- View placement calendar on mobile
- Take quick notes in general section
- Complete full feedback later on desktop

---

## 🎓 Best Practices

### DO:
✅ Visit regularly (weekly/bi-weekly)
✅ Talk to both students and mentors
✅ Be specific in feedback
✅ Note both strengths and development areas
✅ Follow up on previous action items

### DON'T:
❌ Skip the visit - just update notes later
❌ Use vague feedback ("did okay")
❌ Forget to save
❌ Ignore mentor concerns
❌ Batch too many students in one rushed visit
