# PCR Rubric Interpretation Guide for AI Evaluation

## Purpose
This guide provides strict, literal interpretation standards for AI-powered evaluation of HEM3923 PCR submissions. The AI must apply these standards rigorously to ensure fair and accurate assessment.

---

## Core Principles

### 1. LITERAL INTERPRETATION
- **Rule**: All required elements must be **explicitly documented** in the submission
- **NO Assumptions**: Do not assume implied, indirect, or partially present elements
- **NO Generosity**: If an element is not clearly present, it is missing

### 2. EVIDENCE-BASED SCORING
- **Rule**: Every score must be justified with specific quotes from the submission
- **Documentation**: List missing elements explicitly
- **Transparency**: Show exactly what was found and what was missing

### 3. STRICT LEVEL MATCHING
- **Rule**: Match submission to the most accurate level descriptor
- **Best Practice (5)**: Reserved for truly complete, excellent work with ALL elements present
- **Exceeds (4)**: 1 element missing or incomplete
- **Minimum (3)**: 2 elements missing or incomplete
- **Below (2)**: 3+ elements missing or incomplete
- **Not Met (0)**: Critical failures or unsafe practice

---

## Category 1: Case details & Call timings (5 marks)

### Required Elements (ALL must be present for Best Practice)

1. **Anonymised patient demographics**
   - ✅ ACCEPTABLE: "Male, 65 years old", "Female, 42 yo", "M/75"
   - ❌ UNACCEPTABLE: Patient name visible, identifiable information

2. **998 call details/code**
   - ✅ ACCEPTABLE: "Code 3 Cardiac", "998 Priority 1", "Emergency Response Code"
   - ❌ UNACCEPTABLE: Missing code, only "cardiac emergency" (must have explicit code)
   - **NOTE**: Simply stating incident type is NOT sufficient - must have dispatch code

3. **Incident type & location**
   - ✅ ACCEPTABLE: "Chest pain at 123 Main St", "Cardiac emergency, home residence"
   - ❌ UNACCEPTABLE: Location missing, vague ("at home")

4. **Accurate timings of call duration**
   - ✅ ACCEPTABLE: "Call: 14:30, On scene: 14:42, Hospital: 15:15"
   - ✅ ACCEPTABLE: Multiple timestamps showing call progression
   - ❌ UNACCEPTABLE: Only 1-2 times, missing key timestamps, vague ("afternoon")

### Scoring Guide

| Score | Elements Missing | Example |
|-------|------------------|---------|
| 5 | 0 (all present) | All 4 elements clearly documented |
| 4 | 1 missing/incomplete | Missing 998 code OR incomplete timings |
| 3 | 2 missing/incomplete | No code + incomplete location |
| 2 | 3 missing/incomplete | Only demographics + incident type |
| 0 | Critical failure | Patient identifiable OR no timings at all |

## Category 2: Primary Survey & History taking (5 marks)

### Required Elements (ALL must be present for Best Practice)

1. **Patient position**
   - ✅ ACCEPTABLE: "Supine on bed", "Sitting upright in chair", "Prone on floor"
   - ❌ UNACCEPTABLE: Not documented, vague ("comfortable")

2. **Medical/Trauma primary survey (ABCDE)**
   - ✅ ACCEPTABLE: Complete ABCDE with findings for each
     - Airway: Patent/obstructed/protected
     - Breathing: Rate, quality, sounds, SpO2
     - Circulation: Pulse, BP, skin signs
     - Disability: LOC, GCS, pupils
     - Exposure: Injuries, temperature
   - ❌ UNACCEPTABLE: Missing any ABCDE component, incomplete assessment

3. **SAMPLE history (ALL 6 components)**
   - ✅ ACCEPTABLE: S-A-M-P-L-E all documented
     - S: Signs/Symptoms
     - A: Allergies
     - M: Medications
     - P: Past medical history
     - L: Last oral intake
     - E: Events leading to incident
   - ❌ UNACCEPTABLE: Missing any SAMPLE component

4. **OPQRST of chief complaint (ALL 6 components)**
   - ✅ ACCEPTABLE: O-P-Q-R-S-T all documented for pain/complaint
     - O: Onset
     - P: Provocation/Palliation
     - Q: Quality
     - R: Radiation
     - S: Severity (numeric scale)
     - T: Time/Duration
   - ❌ UNACCEPTABLE: Missing any OPQRST component, incomplete pain assessment

### Scoring Guide

| Score | Elements Missing | Example |
|-------|------------------|---------|
| 5 | 0 (all present) | Position + complete ABCDE + complete SAMPLE + complete OPQRST |
| 4 | 1 missing/incomplete | Missing patient position OR incomplete SAMPLE (5/6) |
| 3 | 2 missing/incomplete | No position + incomplete OPQRST |
| 2 | 3 missing/incomplete | Only partial ABCDE + incomplete SAMPLE + no OPQRST |
| 0 | Critical failure | No primary survey OR no history taken |

## Category 3: Secondary Survey & Vital Signs (5 marks)

### Required Elements (ALL must be present for Best Practice)

1. **Minimum 3 sets of vital signs**
   - ✅ ACCEPTABLE: 3+ complete sets with timestamps
     - Must include: BP, HR, RR, SpO2, Temp, GCS
   - ❌ UNACCEPTABLE: <3 sets, incomplete sets (missing values), no timestamps

2. **Accurate use of GCS calculation**
   - ✅ ACCEPTABLE: "GCS 15 (E4 V5 M6)", "GCS 12 (E3 V4 M5)"
   - ❌ UNACCEPTABLE: Only total score, incorrect calculation, "GCS normal"

3. **Accurate use of annotated silhouette**
   - ✅ ACCEPTABLE: Visual body diagram with markings/annotations showing:
     - Injury locations
     - Assessment areas
     - Pain locations
     - Physical findings
   - ❌ UNACCEPTABLE: Text description only (e.g., "chest pain" without diagram)
   - ❌ UNACCEPTABLE: No visual representation, no body map
   - **CRITICAL**: This must be a VISUAL element, not text

4. **Appropriate body systems review**
   - ✅ ACCEPTABLE: Systematic review of multiple systems:
     - Cardiovascular: Heart sounds, pulses, perfusion
     - Respiratory: Breath sounds, chest movement
     - Neurological: Mental status, motor, sensory
     - Gastrointestinal: Abdomen, bowel sounds
     - Musculoskeletal: Movement, deformities
     - Skin: Color, temperature, integrity
   - ❌ UNACCEPTABLE: Single system only
   - ❌ UNACCEPTABLE: "No other systems affected" without documentation
   - ❌ UNACCEPTABLE: Generic "all systems normal"

### Scoring Guide

| Score | Elements Missing | Example |
|-------|------------------|---------|
| 5 | 0 (all present) | 3+ sets + GCS breakdown + visual silhouette + systematic review |
| 4 | 1 missing/incomplete | All present but silhouette not annotated OR only 2 systems reviewed |
| 3 | 2 missing/incomplete | No silhouette + limited systems review (1-2 systems) |
| 2 | 3 missing/incomplete | Only 2 vital sets + no silhouette + no GCS breakdown |
| 0 | Critical failure | No vitals OR use of "NAD"/"N/R" OR no secondary survey |

## Category 4: Interventions & Treatments (inc. Critical Care) (5 marks)

### Required Elements (ALL must be present for Best Practice)

1. **CPG adherence and scope of practice**
   - ✅ ACCEPTABLE: Follows clinical practice guidelines, within paramedic scope
   - ❌ UNACCEPTABLE: Outside scope, contraindicated treatments, unsafe practice

2. **Correct terminology, units, values**
   - ✅ ACCEPTABLE: "Aspirin 300mg PO", "Morphine 5mg IV", "O2 15L via NRB"
   - ❌ UNACCEPTABLE: Trade names, missing units, incorrect values, unclear dosing

3. **ECG interpretation (if applicable)**
   - ✅ ACCEPTABLE: Rate, rhythm, axis, intervals, ST changes, specific diagnosis
     - Example: "Sinus tachycardia 110bpm, ST elevation in II, III, aVF - inferior STEMI"
   - ❌ UNACCEPTABLE: "Normal ECG", "Abnormal ECG", rhythm only without interpretation

4. **Reassessment post-intervention**
   - ✅ ACCEPTABLE: Documented changes after treatment with specific values
     - Example: "Pain reduced from 8/10 to 4/10 after GTN"
     - Example: "SpO2 improved from 88% to 96% on O2"
   - ❌ UNACCEPTABLE: No reassessment, no values, "patient improved"

5. **Rationale for omitted treatments**
   - ✅ ACCEPTABLE: Explains why certain treatments weren't given
     - Example: "GTN withheld due to systolic BP <100mmHg"
     - Example: "Limited to 2 GTN doses due to BP response"
   - ❌ UNACCEPTABLE: No rationale, no explanation for decisions

### Scoring Guide

| Score | Elements Missing | Example |
|-------|------------------|---------|
| 5 | 0 (all present) | CPG-compliant + correct terminology + comprehensive ECG + clear reassessment + rationale documented |
| 4 | 1 missing/incomplete | All interventions appropriate but missing rationale for some decisions |
| 3 | 2 missing/incomplete | Good practice but limited interventions + minimal reassessment |
| 2 | 3 missing/incomplete | Safe but minimal CPG adherence + no ECG interpretation + no reassessment |
| 0 | Critical failure | Unsafe practice OR outside scope OR missing values/units |

## Category 5: Clinical Impression & Contextualization & Documentation (5 marks)

### Required Elements (ALL must be present for Best Practice)

1. **Clinical understanding (specific diagnosis)**
   - ✅ BEST (5): Specific diagnosis - "Acute inferior STEMI", "Diabetic ketoacidosis"
   - ✅ GOOD (4): Probable diagnosis - "Likely acute coronary syndrome", "Probable sepsis"
   - ✅ MINIMUM (3): General problem - "Cardiac emergency", "Respiratory distress"
   - ❌ BELOW (2): Symptoms as diagnosis - "Chest pain", "Shortness of breath"
   - ❌ NOT MET (0): No understanding, layman terms only

2. **Impression-treatment correlation**
   - ✅ ACCEPTABLE: Treatment clearly matches clinical impression
     - STEMI → Aspirin + GTN + Morphine + STEMI alert
     - Hypoglycemia → Glucose administration
   - ❌ UNACCEPTABLE: Disconnect between diagnosis and treatment

3. **Management log quality**
   - ✅ BEST (5): Accurate, logical, chronological with complete details
   - ✅ GOOD (4): Chronological with pertinent information
   - ✅ MINIMUM (3): Chronological but limited detail
   - ❌ BELOW (2): Limited, incomplete, or unclear
   - ❌ NOT MET (0): No management log

4. **Medical terminology usage**
   - ✅ BEST (5): Excellent terminology throughout (ST elevation, inferior leads, etc.)
   - ✅ GOOD (4): Appropriate terminology (cardiac, respiratory, etc.)
   - ✅ MINIMUM (3): Limited medical terms, basic language
   - ❌ BELOW (2): Basic terminology with incorrect values
   - ❌ NOT MET (0): Layman terms only, no medical language

### Scoring Guide

| Score | Quality Level | Example |
|-------|---------------|---------|
| 5 | Best Practice | Specific diagnosis + excellent correlation + accurate logical log + excellent terminology |
| 4 | Exceeds | Probable diagnosis + good correlation + chronological log + appropriate terminology |
| 3 | Minimum | General problem + some correlation + limited log + limited terminology |
| 2 | Below | Symptoms as diagnosis + poor correlation + basic log + basic terminology |
| 0 | Not Met | No understanding + unsafe practice + no log + layman terms only |

## Common Mistakes AI Must Avoid

### ❌ Do NOT Do This:

1. **Assuming Implied Elements**
   - "The silhouette is implied through systematic documentation" → NO
   - "Call code is implied as cardiac emergency" → NO
   - Must be explicitly documented

2. **Being Generous with Missing Elements**
   - If body systems review only states "cardiovascular concern" → This is NOT complete
   - If only 2 vital sign sets → This does NOT meet "minimum 3"
   - If GCS shows only "15" without breakdown → This is NOT "accurate use of GCS calculation"

3. **Overlooking Rubric Requirements**
   - Rubric says "annotated silhouette" → Must see visual diagram
   - Rubric says "998 call code" → Must see explicit code, not just incident type
   - Rubric says "rationale for treatments" → Must see explanations, not just list

4. **Scoring Based on Overall Quality Instead of Specific Criteria**
   - Even if overall work is excellent, missing required elements = lower score
   - Best Practice requires ALL elements, not just good overall performance

## AI Evaluation Checklist

Before assigning Best Practice (5 marks) to any category, verify:

### Category 1: Case details & Call timings
- [ ] Patient anonymised? (check no names/IDs)
- [ ] 998 call code explicitly documented? (not just incident type)
- [ ] Incident type stated?
- [ ] Location documented?
- [ ] Multiple accurate timestamps? (at least 3)

### Category 2: Primary Survey & History
- [ ] Patient position documented?
- [ ] ABCDE all present? (count: must be 5)
- [ ] SAMPLE all present? (count: must be 6)
- [ ] OPQRST all present? (count: must be 6)

### Category 3: Secondary Survey & Vitals
- [ ] 3+ complete vital sign sets? (count sets)
- [ ] GCS breakdown shown? (E_V_M_)
- [ ] Visual annotated silhouette present? (must be visual, not text)
- [ ] Systematic body systems review? (count systems: need 3+)

### Category 4: Interventions & Treatments
- [ ] Follows CPG? (verify guidelines)
- [ ] Correct units/values? (check all medications)
- [ ] ECG interpretation comprehensive? (not just rhythm)
- [ ] Reassessment with specific values? (check for "from X to Y")
- [ ] Rationale for decisions documented? (check explanations)

### Category 5: Clinical Impression
- [ ] Specific diagnosis? (not symptoms)
- [ ] Treatment matches impression? (check correlation)
- [ ] Chronological management log? (check timeline)
- [ ] Medical terminology throughout? (assess language)

## Reporting Requirements

For each category evaluated, AI must provide:

1. **Score**: Numeric value (0, 2, 3, 4, or 5)
2. **Level**: Text description (e.g., "Best Practice", "Exceeds Expected Standard")
3. **Justification**: Detailed explanation with specific evidence
4. **Missing Elements**: Array of any required elements that are missing or incomplete
5. **Evidence**: Direct quotes from submission supporting the score

### Example Output:

```json
{
  "Case details & Call timings": {
    "points": 4,
    "level": "Exceeds Expected Standard",
    "justification": "The submission includes anonymised demographics ('Male, 65 years old'), incident type and location ('Chest pain, cardiac emergency at Home residence, 123 Main Street'), and accurate timings ('Call received: 14:30, On scene: 14:42, Hospital arrival: 15:15'). However, the explicit 998 call code is not documented.",
    "missingElements": ["998 call details/code"],
    "evidence": "Direct quotes: 'Male, 65 years old', 'Call received: 14:30, On scene: 14:42, Hospital arrival: 15:15'"
  }
}
```

## Confidence Rating Guidelines

Rate confidence based on:

- **0.9-1.0**: Submission is clear, complete, unambiguous
- **0.7-0.8**: Submission is mostly clear with minor ambiguities
- **0.5-0.6**: Submission has significant ambiguities or unclear sections
- **0.3-0.4**: Submission is difficult to assess or very incomplete
- **0.0-0.2**: Submission is severely lacking or impossible to assess accurately

## Final Reminder

**The AI evaluator must be STRICT, LITERAL, and EVIDENCE-BASED.**

- Do not assume
- Do not be generous
- Do not overlook missing elements
- Count all required elements
- Quote specific evidence
- Match to the most accurate level
- Reserve Best Practice for truly complete work

This ensures fair, consistent, and rigorous evaluation that helps students understand exactly what is required.
