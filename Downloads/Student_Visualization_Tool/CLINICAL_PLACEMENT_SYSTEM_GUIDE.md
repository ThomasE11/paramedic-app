# Clinical Placement Tracking System - Complete Guide

## Overview

The Clinical Placement Tracking System is a comprehensive solution for managing student clinical placements, conducting site visits, and maintaining detailed student feedback records that automatically integrate with student profiles.

---

## 🎯 Key Features

### 1. **AI-Powered Schedule Upload**
- Upload placement schedules in any format (text, copy-paste from Excel, etc.)
- AI automatically parses student names, sites, dates, shifts, and mentors
- Automatic student matching with database
- Site creation/matching with conflict detection

### 2. **Visual Placement Calendar**
- Weekly view of all clinical placements
- Group by site or by student
- Quick overview of who's where at any time
- Filter by module, week, or site type

### 3. **Site Visit Documentation**
- Comprehensive site visit form
- Multi-student feedback in single visit
- Automatic note creation in student profiles
- Rich assessment capabilities

### 4. **Student Profile Integration**
- All site visit notes automatically appear in student profiles
- Searchable and categorized as "site_visit"
- Includes performance, strengths, areas for development
- Skills tracking and competency assessment

## 🚀 Workflow Guide

### Step 1: Upload Placement Schedule

#### Using the Upload Feature

1. Click on **"Upload Schedule"** tab
2. Paste your placement schedule in any format

**Example Format:**
```
Week 3-6 (Jan 15 - Feb 12, 2025):
- Fatima Al Mazrouei (H20230101) at Al Ain Hospital - Emergency Department, Day shift, Mentor: Dr. Ahmed
- Mohammed Hassan (H20230102) at Sheikh Khalifa Medical City - ICU, Night shift, Mentor: Nurse Sarah

Week 7-10 (Feb 13 - Mar 12, 2025):
- Fatima Al Mazrouei (H20230101) at Abu Dhabi Ambulance Station 1, Rotating shifts
- Mohammed Hassan (H20230102) at Community Health Center - Primary Care
```

3. Click **"Parse Schedule"** (AI processes it)
4. Review the parsed placements:
   - ✅ **Green** = Student matched in database
   - ⚠️ **Orange** = Student not found
5. The AI will:
   - Match students by ID or name
   - Create new sites if they don't exist
   - Detect date formats automatically
   - Extract mentors, shifts, departments

6. Click **"Create X Placements"** to finalize

**Supported Schedule Formats:**
- Bullet lists
- Tables (copy-pasted)
- Plain text with dates
- CSV-style formatting
- Any human-readable format!

---

### Step 2: View Placement Calendar

1. Click on **"Clinical Placements"** tab
2. Use the week navigation (Previous/Today/Next)
3. Toggle between **By Site** or **By Student** view

**By Site View:**
- See all students at each location
- Quick "Site Visit" button for each site

**By Student View:**
- See all placements for each student

**Placement Information Shown:**

- Shift type (day/night/rotating)
- Mentor name
- Status (scheduled/in_progress/completed/cancelled)

### Step 3: Conduct Site Visit & Record Notes

This is the **KEY FEATURE** you requested!

#### Starting a Site Visit

1. In the **Clinical Placements** calendar
2. Find the site you're visiting
3. Click **"Site Visit"** button

#### Filling Out the Site Visit Form

**Visit Details Section:**
- **Visit Date** * (Required)
- **Visit Time** (Optional but recommended)
- **Duration** (in minutes)
- **Purpose** (e.g., "Routine check-in", "Performance review")
- **General Notes** - Overall observations
- **Mentor Feedback** - Feedback from site supervisors
- **Areas of Concern** - Any issues identified
- **Recommendations** - Next steps or improvements
- **Follow-up Required** - Check if another visit needed
- **Follow-up Date** - When to revisit

**Student Feedback Section:**

For each student present at the site, you can record:

1. **Performance Assessment**
   - Excellent / Good / Satisfactory / Needs Improvement / Unsatisfactory

2. **Attendance & Hours**
   - Attendance Status: Present / Absent / Late / Left Early
   - Hours Completed: Actual clinical hours logged

3. **Qualitative Feedback**
   - **Strengths** ⭐ - What did they excel at?
   - **Areas for Development** 📈 - What needs improvement?
   - **Mentor Comments** - Direct feedback from clinical mentor
   - **Student Reflection** - Student's own thoughts (optional)

4. **Concerns & Actions**
   - **Concerns** ⚠️ - Specific issues or risks
   - **Action Items** ✅ - Next steps and follow-ups

5. **Skills Assessment**
   - Add skills assessed (e.g., "IV Insertion", "Patient Assessment")
   - Type skill name and press Enter or click +
   - Remove skills with X button

6. **Grade** (Optional)
   - Assign a grade if applicable

#### What Happens When You Save?

1. **Site Visit Record Created**
   - Complete visit logged with all details
   - Searchable and reportable
   - Linked to site and students

2. **Student Profile Notes AUTO-CREATED** ⭐⭐⭐
   - Each student gets a comprehensive note in their profile
   - Title: "Site Visit - [Site Name]"
   - Category: "site_visit" (searchable)
   - Contents include:
     - Visit date
     - Performance rating
     - Strengths
     - Areas for development
     - Mentor comments
     - Student reflection
     - Skills assessed
     - Hours completed
     - Attendance status
     - Grade (if given)
     - Concerns
     - Action items

3. **Notes Are Immediately Visible**
   - Go to Students section
   - Click on any student who was in the visit
   - See the note in their profile
   - Notes are timestamped and traceable

## 💡 Pro Tips

### For Efficient Site Visits

1. **Pre-fill Visit Details**
   - Visit date, time, duration are auto-set to current
   - Just update if different

2. **Use Skills Templates**
   - Common skills for your program:
     - Clinical skills: "IV Insertion", "Blood Draw", "Vital Signs"
     - Assessment: "Patient Interview", "Physical Exam"
     - Emergency: "CPR", "Defibrillation", "Airway Management"
     - Documentation: "Patient Charting", "Incident Reports"

3. **Quick Feedback Patterns**
   - Save common feedback phrases you use
   - Copy/paste and customize for each student

4. **Take Photos/Documents**
   - Attachment field supports JSON
   - Can link to external files if needed

### For Schedule Management

1. **Test Your Schedule First**
   - Try parsing with 2-3 entries first
   - Verify AI understands your format
   - Then upload full schedule

2. **Standardize Your Format**
   - Once you find a format that works well
   - Use it consistently for future semesters

3. **Student ID Matching**
   - Always include student IDs in schedule
   - More reliable than name matching
   - Format: H20230101 (your institution's format)

4. **Site Naming Consistency**
   - Use same name each time for a site
   - e.g., "Al Ain Hospital" not "Al-Ain Hospital Emergency"
   - System will recognize and link them

## 🔧 API Endpoints Reference

### Placement Sites
- `GET /api/placements/sites` - List all sites
- `POST /api/placements/sites` - Create new site
- `PUT /api/placements/sites` - Update site
- `DELETE /api/placements/sites?id={id}` - Deactivate site

### Placement Assignments
- `GET /api/placements/assignments` - List assignments
  - Query params: `studentId`, `siteId`, `moduleId`, `status`, `startDate`, `endDate`
- `POST /api/placements/assignments` - Create assignment
- `PUT /api/placements/assignments` - Update assignment
- `DELETE /api/placements/assignments?id={id}` - Cancel assignment

### Site Visits
- `GET /api/placements/visits` - List visits
  - Query params: `siteId`, `studentId`, `visitId`, `include=notes`
- `POST /api/placements/visits` - Create visit with student notes
- `PUT /api/placements/visits` - Update visit
- `DELETE /api/placements/visits?id={id}` - Delete visit

### Site Visit Notes
- `GET /api/placements/visits/notes` - List notes
  - Query params: `visitId`, `studentId`, `noteId`
- `POST /api/placements/visits/notes` - Add note to visit
- `PUT /api/placements/visits/notes` - Update note
- `DELETE /api/placements/visits/notes?id={id}` - Delete note

### AI Schedule Parsing
- `POST /api/placements/parse-schedule` - Parse schedule text
  - Body: `{ scheduleText, moduleId }`
- `PUT /api/placements/parse-schedule` - Create assignments from parsed
  - Body: `{ placements[], moduleId }`

## 🗄️ Database Schema

### PlacementSite
Clinical placement locations

**Fields:**
- `id` - Unique identifier
- `name` - Site name (e.g., "Al Ain Hospital")
- `type` - hospital | ambulance_station | community_clinic | primary_care
- `address`, `city` - Location details
- `contactPerson`, `contactEmail`, `contactPhone` - Contact info
- `capacity` - Max students
- `department` - Specific department
- `specialization` - Area of focus
- `facilities` - JSON equipment/facilities list
- `notes` - General notes
- `isActive` - Active status

### PlacementAssignment
Student-to-site assignments

**Fields:**
- `id` - Unique identifier
- `studentId` - Reference to student
- `siteId` - Reference to site
- `moduleId` - Reference to module (optional)
- `startDate`, `endDate` - Placement period
- `weekNumber` - Week of rotation
- `rotationType` - clinical | observational | community
- `shift` - day | night | rotating
- `mentor` - Supervisor name
- `objectives` - Learning objectives array
- `status` - scheduled | in_progress | completed | cancelled
- `notes` - Additional notes
- `createdBy` - Who created it

### SiteVisit
Visit documentation

**Fields:**
- `id` - Unique identifier
- `siteId` - Site visited
- `visitDate` - When visited
- `visitTime` - Time of visit
- `visitedBy` - Who conducted visit
- `purpose` - Purpose of visit
- `duration` - How long (minutes)
- `generalNotes` - Overall observations
- `mentorFeedback` - From site staff
- `areasOfConcern` - Issues identified
- `recommendations` - Suggestions
- `followUpRequired` - Boolean
- `followUpDate` - When to return
- `attachments` - JSON file references

### SiteVisitNote
Individual student feedback

**Fields:**
- `id` - Unique identifier
- `visitId` - Parent visit
- `studentId` - Student being assessed
- `performance` - Rating
- `competencies` - JSON competency assessment
- `strengths` - Text
- `areasForDevelopment` - Text
- `mentorComments` - Text
- `studentReflection` - Text
- `grade` - Optional grade
- `skillsAssessed` - Array of skills
- `attendanceStatus` - present | absent | late | left_early
- `hoursCompleted` - Float
- `concerns` - Text
- `actionItems` - Text

## 🚨 Important Notes

### Automatic Student Profile Integration

**This is the core feature you requested!**

Every time you save a site visit:
1. A `SiteVisit` record is created
2. For each student, a `SiteVisitNote` is created
3. **Automatically**, a `Note` is created in the student's profile
4. This note contains ALL the feedback you entered
5. It's categorized as "site_visit" for easy filtering
6. It's timestamped and linked to you as the creator

**To view student notes:**
1. Go to Students section
2. Click on a student
3. Scroll to their notes
4. You'll see all site visit notes with full details

### Conflict Detection

The system automatically detects:
- Overlapping student assignments
- Site capacity violations
- Module scheduling conflicts

### Performance Considerations

- Placements are indexed by date for fast querying
- Site visits load only recent ones by default
- Student notes are lazy-loaded

### Security

- All endpoints require authentication
- Role-based access (instructors only)
- Student data privacy maintained

## 📞 Support & Troubleshooting

### Common Issues

**Q: AI didn't match my students**
A: Ensure student IDs are in database and format matches

**Q: Site wasn't created**
A: Check site name in parsed data, may need manual creation

**Q: Notes didn't appear in student profile**
A: Refresh the student page, check category filter

**Q: Can't see placement calendar**
A: Verify you have instructor role

### Error Messages

- `"Student has conflicting placement"` - Check dates, student may already be assigned
- `"Unauthorized"` - Login required
- `"Site ID required"` - Form validation error
