## Trends
- CATEGORY UNILATERALITY GRADIENT: respiratory + cardiac-arrest cases are the MOST dynamic (asthma/COPD/anaphylaxis/tension-PTX/cardiac-arrest all have deep protocols + pathology modifiers + staged timelines + hard-coded guards). Toxicology, metabolic (DKA/hyperK), obstetric, environmental (heat/drowning), burns, psychiatric, paediatric-sepsis/croup are the MOST unilateral — almost all lack a deep protocol AND fall through applyDeterioration's generic 'else' branch (pulse +2 / SpO2 -1) which is clinically inverted (it pushes bradycardic/shocked patients the WRONG way and is far too slow).
- THE TWO UNIVERSALLY-MISSING BRANCHES for the ~30 protocol-less conditions are (4) contraindicated-harm and (5) untreated-deterioration. (4) is missing because findProtocol returns null => no contraindicatedTreatments list => the 12-pt StudentPanel penalty channel is dead AND there is usually no matching cross-system vital guard. (5) is missing because applyDeterioration reads getCaseDeteriorationTimeline (keyed on subcategory/alias/category) — it does NOT read the protocol's deteriorationRate field, and does NOT read the case's own vitalSignsProgression.deterioration block in the staged path. So a new protocol ALONE never produces an untreated clock; a CASE_DETERIORATION_TIMELINES entry or SUBCATEGORY_ALIAS is always also required.
- FINDPROTOCOL SUBSTRING/FALLBACK is the single biggest structural defect: it only has cardiac category fallbacks (arrest, stem/nstemi/acs). Every ACS case authored with subcategory 'acute-coronary-syndrome' (litfl-019, y2-002, litfl-001) returns NULL on the protocol layer (string lacks 'acs'/'stem' substring) yet clinicalRealism aliases it to stem-ANTERIOR — so they get anterior pathology modifiers but no deep-protocol synergies/contraindications. Many trauma cases authored as subcategory 'chest-trauma' (resp-006, trauma-003/004/005/006) never reach the perfect pneumothorax-tension/open/tamponade protocols. A small set of findProtocol/alias edits fixes a dozen cases at once.
- BORROWED/MIS-FIT PROTOCOL is a recurring false-cover: (a) inferior/RV STEMI (cardiac-007, cardiac-ecg-001) and LBBB-STEMI (cardiac-008) borrow stem-ANTERIOR whose GTN-permissive synergies and empty/anterior contraindication lists are physiologically wrong (GTN is contraindicated in RV infarct); (b) hypothermic/electrical asystolic arrest (cardiac-014, burn-002) borrow the SHOCKABLE cardiac-arrest protocol whose defib-centric synergies mis-fit non-shockable rhythms; (c) WORST — empty-subcategory cases (psych-001 panic, fall-001) match the FIRST protocol (asthma) via p.condition.includes('')===true, applying spurious asthma beta-blocker contraindication penalties to a panic/fall patient.
- MISSING CATALOG TREATMENTS block the correct/contraindicated branches entirely for several high-acuity conditions: NO insulin (DKA bolus-before-fluid error + hyperK shift therapy unrepresentable), NO antibiotics/dexamethasone (meningitis/sepsis/pneumonia 'antibiotics within 1h' cannot be given or rewarded), NO acetylcysteine/NAC (paracetamol OD has no correct answer), NO pralidoxime (organophosphate), NO pacing_transcutaneous / atropine_500mcg / calcium_gluconate (the three bradycardia/heart-block cases reference IDs that don't exist), NO furosemide / mannitol (only mannitol_20) / occlusive_dressing_3sided / cooling / cling_film / haloperidol+promethazine / breathing_coaching / emesis-induction. Protocols that reference a non-existent ID silently no-op (no compile error).
- SCORING-ONLY HALF-LOOPS are pervasive even in the GOOD cases: contraindicated drugs frequently burn the 12-pt penalty but move no vital because no matching guard exists (adenosine-in-AF cardiac-003; GTN/aspirin-in-stroke at normotensive BP neuro-001/y2-003; oral-glucose-aspiration metab-001). Conversely some real vital harms (GTN inferior-STEMI chain, opioid RR<=10 guard) carry NO scoring penalty because the borrowed/absent protocol lists nothing. Harm needs BOTH a contraindicatedTreatments entry (scoring) AND a guard/cross-system chain (physiology) to be a full bidirectional loop.
- GENUINELY-NON-PHARMACOLOGIC cases are a distinct cluster that the vital/drug engine cannot model and should be scored on checklists instead: assessment/disposition (cardiac-005 stable angina, cardiac-010/general-001 syncope, cardiac-ecg-002 Wellens 'don't stress-test', y2-006 TIA 'don't lower BP'), procedural (y1-006 normal delivery), and multi-patient (multi-001 MCI runs ONE aggregate PatientState so per-patient dynamics are impossible). For these the fix is engine/data-shaped (suppress false deterioration, add decision actions, or instantiate per-patient state), not a SeverityProtocol.
- LOW-ACUITY STABLE cases are FALSELY destabilised by the generic clock: stable angina, recovered syncope, sinus tach (y1-013), isolated Colles/whiplash (y1-010/011), minor scald/Colles — all 'stable' presentations get generic category drift (cardiac SBP fall / trauma SBP crash) that is clinically wrong (a recovered vasovagal faint should not silently hypotense). For these the correct fix is to FLATTEN/no-op the untreated timeline, the opposite of the high-acuity fix.

## Engine changes still pending
1. DETERIORATION IS THE #1 ENGINE DEPENDENCY: applyDeterioration (dynamicTreatmentEngine.ts ~L1222) reads getCaseDeteriorationTimeline (keyed subcategory->alias->category in clinicalRealism) and IGNORES the protocol's deteriorationRate field entirely. So EVERY new protocol MUST be paired with a CASE_DETERIORATION_TIMELINES entry (or a SUBCATEGORY_ALIAS to an existing one) or it produces ZERO untreated decline. The generic fallback (L1299-1338) only has respiratory/cardiac/trauma branches; metabolic/toxicology/pediatric/obstetric/environmental/burns/psychiatric all hit the bland 'else' (pulse+2/SpO2-1) which is clinically inverted for shocked/bradycardic patients — these categories need explicit timelines, not just protocols.
2. findProtocol (treatmentProtocols.ts L1600) needs broader matching: (a) consult SUBCATEGORY_ALIAS (currently only clinicalRealism's getPathologyModifiers/getCaseDeteriorationTimeline use it — findProtocol does NOT), so 'acute-coronary-syndrome'/'aflutter'/'tia'/'asystole'/'extremity-trauma'/'cerebrovascular-emergency' resolve to their deep protocols; (b) add category fallbacks beyond cardiac (e.g. environmental+arrest, trauma+chest-findings -> pneumothorax-tension/open/tamponade/massive-hemorrhage routed by dx/findings); (c) GUARD against the empty-subcategory bug: p.condition.includes('') is always true so an empty subcategory matches the FIRST protocol (asthma) — add `if (!sub) return null` or require non-empty before the substring pass (fixes psych-001, fall-001 borrowing asthma).
3. CONTRAINDICATED-HARM needs BOTH halves: a contraindicatedTreatments entry only burns the 12-pt StudentPanel score; to actually worsen vitals you must add a matching guard in applyStandardTreatment or a chain in applyCrossSystemPhysiology. New guards/chains required: fluids in PE (over-distend RV -> drop BP/SpO2), fluids in neurogenic shock (over-fluid -> pulmonary oedema/RR drop), adenosine-in-AF/flutter (transient AV block/hypotension), AV-nodal-blocker/adenosine in CHB/bradycardia (abolish escape -> standstill), paracetamol in heat-stroke (zero the antipyretic temp-drop via a 0.0 PATHOLOGY_MODIFIER), opioid-in-hypotensive-shock (worsen BP/RR when SBP<100 regardless of RR), salbutamol-in-cardiac-asthma de-reward, needle_decompression-in-tamponade/haemothorax collapsed to ~0.
4. Broaden existing guards that are too narrowly keyed: (a) the SBP<90 GTN guard (L550) and the inferior-STEMI GTN chain (L1425) are keyed to treatment.id==='gtn_spray' only — also match 'gtn_sl'/'nitroglycerin' (litfl-001 false-negative); the SBP<90 threshold is a near-miss at exactly 90 (cardiac-001) — consider <100 OR a hypotensive-anterior-STEMI contraindication. (b) the heart-failure fluid SpO2 chain (L1382) is gated on normalisedSub.includes('heart failure'/'chf'/'pulmonary oedema') — broaden to ALSO fire on findings/dx containing 'crackles'/'pulmonary oedema' regardless of subcategory so lbbb-stemi (cardiac-008) and electrolyte-emergency+overload (litfl-003) get real fluid harm. (c) the cerebral-oedema osmotherapy +GCS chain checks sub.includes('stroke'/'head'/'tbi'/'intracranial') — add 'cerebrovascular'/'sah'/'subarachnoid' (litfl-012). (d) tighten the opioid hard-guard RR<=10 -> RR<=12 to match the quality rule (lands harm at trauma-009 RR 18 once it falls; trauma-002).
5. responseCeilings.partialCeiling/fullCeiling are DEFINED BUT NOT CONSUMED (getEffectiveResponseCeiling has no caller; applyStandardTreatment never reads them). To make 'essential-only caps at partialCeiling' actually bite, either wire getEffectiveResponseCeiling into the effect math, or (cheaper, matches current behaviour) keep relying on synergy TIERS + the spo2Ceiling from clinicalRealism.getTreatmentEffectivenessMultiplier — meaning every new protocol must encode its partial-vs-full distinction as a low-tier (essential pair ~1.4) + high-tier (full bundle ~1.8-2.5) synergy, NOT via the ceiling numbers. Fill ceilings sensibly for forward-compat regardless.
6. Wire the engine to consume the case's authored vitalSignsProgression.deterioration block: in the staged path (L1271) only initial BP is used as a baseline for deltas; many cases (general-001, y2-004, litfl-003, burn cases, panic cases) hand-author a full .deterioration vitals snapshot that is currently DEAD DATA. A generic 'if no timeline key, interpolate toward vitalSignsProgression.deterioration over N minutes' fallback would rescue dozens of cases without bespoke timelines.
7. determineSeverityFromVitals coverage: ensure new protocols' typicalVitals bands are tuned so the case's INITIAL vitals fall cleanly inside exactly one level (SpO2 +3 / GCS +3 dominate). Specifically fix the hypoglycaemia 'moderate' gap (GCS 13 matches neither mild nor severe) and verify anaphylaxis y1-015 lands 'moderate' (SpO2 93) vs life-threatening as intended.
8. catalog integrity: there is NO compile-time check that treatment ids exist, so a typo silently no-ops. Add a dev-time assertion/test that every id referenced in TREATMENT_PROTOCOLS (essential/optimal/beneficial/contraindicated/synergy.treatments/positionEffect.positionId) and in every PATHOLOGY_MODIFIER exists in TREATMENTS[] — this would have caught the dead 'pacing_transcutaneous'/'calcium_gluconate'/'atropine_500mcg'/'mannitol'/'furosemide' references the case findings rely on.
9. Defibrillation depth (handleDefibrillation): add a shock-COUNT branch for refractory VF (cardiac-013) — after 3-5 failed shocks, plain repeat shock yields diminishing ROSC unless amiodarone-2nd-dose/Mg/DSED added; add a post-ROSC stage so y2-009's post-arrest care diverges; add 'defibrillation' to a non-shockable variant's contraindicatedTreatments so the 12-pt penalty matches the existing shock-on-asystole adverse event (cardiac-002/burn-002/cardiac-014).
10. MCI (multi-001) is structurally out-of-model: applyDynamicTreatment runs ONE PatientState. A real fix instantiates each mci[] roster entry as its own PatientState+protocol+timeline; until then route MCI grading to studentChecklist only.
